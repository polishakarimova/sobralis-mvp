# Sobralis UI Rules

## Общий принцип

UI должен выглядеть как красивая invitation-система, а не как CRM.

Сначала — эстетика события.  
Потом — функции организатора.

## Карточки

Все карточки:

- мягкий radius;
- молочный фон;
- тонкая рамка;
- мягкая тень;
- много внутреннего воздуха;
- без жёстких таблиц;
- без агрессивных контрастов.

## Главная карточка приглашения

Обязательная структура:

- label: «приглашение»
- title: название события
- description
- дата
- время
- место
- места
- гости
- CTA «Я иду»
- статус «Событие собрано»

Главная карточка должна выглядеть как приглашение, а не форма.

## Организаторские чипы

Показывать отдельными мягкими карточками:

- Места — 6 / 12
- Оплаты — 10 из 12 оплачено
- Состав — 16 гостей
- Ожидание — 2 человека
- Напоминание — За день
- Ссылка-приглашение — Скопирована

## Кнопки

Primary:

- шалфейный фон;
- молочный текст;
- мягкий radius;
- hover: лёгкий подъём и тень.

Secondary:

- молочный фон;
- тонкая рамка;
- графитовый текст.

Warm:

- карамельный фон;
- молочный текст.

## Иконки

Иконки должны быть:

- line icons;
- stroke-width около 1.5;
- спокойные;
- единый стиль;
- без stock-looking набора.

Иконки:

- календарь;
- время;
- место;
- гости;
- стул/место;
- оплата;
- уведомление;
- ссылка;
- галочка;
- ожидание.

## Анимации

Допустимые:

- мягкий hover;
- subtle shadow;
- progress fill;
- status transition;
- smooth copy feedback.

Запрещены:

- резкие bounce-анимации;
- агрессивное движение;
- flashy эффекты;
- confetti everywhere.

## Фоны

Фон:

- молочный;
- лёгкий radial-gradient;
- subtle grain;
- тонкие декоративные дуги;
- мягкие круги.

Не делать:

- цветочный паттерн;
- свадебные рамки;
- много листьев;
- яркий декоративный фон.

## Адаптив

Mobile:

- одна колонка;
- карточка на всю ширину с отступами;
- чипы под карточкой;
- логотип не должен ломаться;
- заголовки не должны быть огромными.
